# Tickets — Transparent (composite over the blueprint)

Each ticket is generated **isolated on a transparent background** (alpha) so it can be
layered over `background-blueprint.md` in Canva. Tickets carry **no background and no cast
ground shadow** — only the ticket (add the drop shadow in Canva to match the blueprint).

Prompts are written **exhaustively** — geometry, material, colour (with hex), border,
graphic, typeface, exact text and placement, finish, and explicit **exclusions** — so the
model leaves nothing to chance. Every ticket is styled **true to its year**.

Shared ticket geometry (identical wording in each prompt): one horizontal event ticket,
about 2.2× wider than tall, filling ~82% of the frame width, centred, rotated ~6°
counter-clockwise, rounded corners; a vertical dashed perforation of small dots runs
top-to-bottom about three-quarters of the way to the right, with a small semicircular notch
cut into the top and bottom edge where it meets them, separating a narrow right-hand stub
from the wider main body.

**Output for every ticket:** a **square image 1024×1024 with a fully transparent
background**, ticket centred and isolated. In ChatGPT, if it returns a solid backing, reply
"make the background fully transparent (alpha), keep only the ticket."
